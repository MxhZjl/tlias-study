5\.数据库当中varchar和text的区别

选型建议（新手直接套用）

优先用 VARCHAR：如果文本长度≤5000 字符（日常大部分场景），选 VARCHAR，性能更好、更灵活；

&#x20;选 TEXT 的情况：文本长度 > 5000 字符，或不确定长度但大概率很长（如文章、详情）；

&#x20;特殊场景：如果文本超 65535 字符（比如几万字的文档），直接用 MEDIUMTEXT，不用纠结 VARCHAR。

&#x20;VARCHAR 和 TEXT 都支持任意字符（中文 / 英文），核心区别是存储上限、存储位置、默认值 / 索引规则；

&#x20;短 / 中等长度文本（<5000 字符）用 VARCHAR（性能优、支持默认值），长文本用 TEXT（容量大、不占行长度）；

&#x20;TEXT 建索引必须指定长度，且 MySQL 中 TEXT 不支持默认值，这是新手最容易踩的坑。
